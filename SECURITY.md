# 安全策略

## 支持的版本

目前仅支持以下版本的安全更新：

| 版本 | 支持状态 |
| --- | --- |
| 0.1.x | ✅ 支持 |
| < 0.1.0 | ❌ 不支持 |

## 报告安全漏洞

如果你发现了安全漏洞，请**不要**通过公开的 Issue 报告。

### 报告方式

1. **GitHub 安全公告**（推荐）
   - 访问项目的 [Security Advisories](https://github.com/yourusername/stm32h750-display/security/advisories)
   - 点击 "Report a vulnerability"
   - 填写漏洞详情

2. **邮件报告**
   - 发送邮件至项目维护者
   - 主题：[Security] 漏洞描述

### 报告内容

请包含以下信息：

- 漏洞类型（例如：缓冲区溢出、权限提升等）
- 影响的版本
- 复现步骤
- 概念验证代码（如果有）
- 可能的影响范围
- 建议的修复方案（如果有）

### 响应时间

- **确认收到**：2 个工作日内
- **初步评估**：7 个工作日内
- **修复时间**：根据漏洞严重程度决定

### 漏洞处理流程

1. **确认**：维护者确认漏洞存在
2. **评估**：评估漏洞严重程度和影响范围
3. **修复**：开发修复补丁
4. **测试**：验证修复有效且不引入新问题
5. **发布**：发布安全更新版本
6. **公告**：发布安全公告说明漏洞详情

### 披露政策

- 我们遵循**负责任披露**原则
- 在修复发布前，请勿公开披露漏洞细节
- 我们会在修复发布后公开致谢报告者（如果你愿意）

## 安全最佳实践

使用本项目时，请注意：

### 硬件安全

- 确保硬件连接正确，避免短路
- 使用适当的电源供应
- 在开发环境中测试后再部署到生产环境

### 固件安全

- 定期更新到最新版本
- 不要在生产环境中使用调试版本
- 保护好调试接口（ST-Link）

### 依赖安全

- 定期更新依赖项
- 使用 `cargo audit` 检查依赖漏洞

```bash
cargo install cargo-audit
cargo audit
```
