# 硬件 SPI + DMA 驱动 ILI9341 设计文档

## 概述

将当前软件 SPI 驱动替换为 STM32H7 硬件 SPI2 + DMA 实现，大幅提升屏幕刷新速度。

## 引脚映射

| 功能 | GPIO | SPI2 复用功能 |
|------|------|--------------|
| SCK  | PB13 | AF5 |
| MISO | PB14 | AF5 |
| MOSI | PB15 | AF5 |
| CS   | PB12 | GPIO（软件控制） |
| DC   | PB1  | GPIO（软件控制） |
| BLK  | PB0  | GPIO（软件控制） |

## 架构设计

### 1. 模块结构

```
src/
├── main.rs         # 初始化 SPI2 和 DMA，创建 Display 实例
└── display.rs      # DisplaySpi 结构体改为使用硬件 SPI
```

### 2. DMA 配置

- **DMA**: DMA2 Stream1 Channel0 (SPI2_TX)
- **模式**: 内存到外设
- **数据宽度**: 16位 (RGB565)
- **优先级**: 高

### 3. 渐进式实现策略

#### 阶段 1: 基础硬件 SPI（无 DMA）
1. 配置 SPI2 外设
2. 实现 8 位数据传输
3. 串口调试验证
4. 屏幕初始化和基本显示

#### 阶段 2: 添加 DMA 支持
1. 配置 DMA2
2. 实现 DMA 传输函数
3. fill_rect 使用 DMA
4. 性能测试

## SPI 配置参数

- **时钟**: 400 MHz 系统时钟 / 4 = 100 MHz SPI 时钟
- **模式**: Mode 0 (CPOL=0, CPHA=0)
- **数据大小**: 8位（初始化）/ 16位（像素数据）
- **位序**: MSB first

## 预期性能提升

| 操作 | 软件 SPI | 硬件 SPI | 硬件 SPI + DMA |
|------|----------|----------|----------------|
| 全屏刷新 | ~200ms | ~50ms | ~10ms |
| CPU 占用 | 100% | ~80% | <5% |

## 调试策略

1. 每个初始化步骤后通过串口输出状态
2. LED 快闪 10 次表示成功到达某阶段
3. 使用 `connect-under-reset` 烧录避免启动时序问题

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| SPI 挂起 | 渐进式实现，先基础后 DMA |
| DMA 配置复杂 | 使用 HAL 抽象，逐步验证 |
| 时钟配置问题 | 保守使用 96MHz 系统时钟 |
